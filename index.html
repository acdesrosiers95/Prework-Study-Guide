HTML

  • HTML stands for Hypertext Markup Language
  • The head element contains information about the webpage.
  • The body element represent the visible content shown to the user
  • Elements that are nested inside the other elements are call child elements
  • The Elements that hold all the child elements are call Parent elements
  • An element refers to the entire element from opening tag to closing tag
  • Tags refer only to what appears inside the andle brackets

CSS

  • CSS stands for Cascading style sheets
  • A margin indicated how much space we want around the outside of an element.
  • A padding indicated how much space we want around the content inside and element.
  • A selector defines the element or attributes to which the rules, or delcarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • To apply a rule to a ll the elements visible on the page we use a special wild card symbol (*).
  • When creating an external CSS file we need to link it to the HTML file with HTML ("link") element.

GIT

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Repository("repo") is a folder for your project that contains all of the files and their revison history. every project on Github is called a Repository.
  • Branch is a workspace where you can make changes that dont affect the live site.
  • Markdown (.md) is the way content files are formatted in Github.
  • Commit changes saves a new version of a file you're working on. When ever you add, edit, or delete a file, you're making a commit.
  • Pull Request ("PR") is asking for changes on a branch to merge into another branch.
  • Merge is when a PR request is approved, changes are "Pulled in" or merged inot main branch.
  • Issue is a way to report new task or content fixes.
